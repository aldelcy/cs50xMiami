Cohort #10 (Fall 2019)

CS50x Miami is The Idea Center @ Miami Dade College's adaptation of CS50, Harvard University's introduction to the intellectual enterprises of computer science, and the art of programming, for MDC students and the South Florida community.

But what does that mean?

Well, CS50x Miami is a course that teaches you how to design and implement solutions to problems. But more than that, it teaches you how to think more critically, more methodically, more algorithmically, and more computationally, if you will.

In fact, computer science itself isn't really about computers (or programming for that matter), it's really about information. How do you think about it? How do you represent it? And with what methods or algorithms can you process it?

So, we'll first learn how to program with Scratch, a graphical programming language via which we'll explore some fundamental programming constructs, by dragging and dropping puzzle pieces.

We'll then quickly transition to a more traditional text-based language called C that's actually been around quite a while. And as such, it doesn't come with that many features out of the box, so to speak. So anything you want the computer to do, you're going to have to teach it to do yourself; from the simplest of algorithms, to the most sophisticated.

Goals

The problems we'll solve in this course are inspired by real-world domains: cryptography, finance, forensics, gaming, and more.

Over the course of CS50x Miami, you'll be part of a global community solving those same problems, surrounded by classmates and so many others on CS50x Miami's team.

But the most successful outcome ahead is to do something, ultimately, that we've not taught you. Indeed, what ultimately matters in this course is not so much where you end up relatively to your classmates, but where you end up relative to yourself when you began.

And when you do end up there, will you be able to say proudly, "I took CS50".

Outcomes

Upon successful completion of the course, students will be able to...

  1. Analyze a problem and craft an appropriate algorithmic solution.
  2. Understand the basic syntax of a programming language (C, Python) so that they can use that language to solve problems.
  3. Design and write programs which utilize the following control structures: selection structures, loop structures, and functions.
  4. Demonstrate the ability to effictively debug a program.
  5. Utilize good program documentation practices in their program code.
  6. Design a program given specific data storage and efficiency requirements.
  7. Implement and use data structures such as stacks, queues, linked lists, trees, and tries in a computer program.
  8. Analyze different methods of meeting programming objectives and use Big-O evaluation methods to identify the relative efficiencies of different solutions.
  9. Apply basic object-oriented programming skills to write and debug programs in an O-O language (Python).
  10. Understand the concepts behind the Model View Controller (MVC) design pattern.
  11. Understand the Client-Server model and the HyperText Transfer Protocol (HTTP).
  12. Design and implement a Website which utilizes HTML, CSS, JavaScript, and Ajax.
  13. Be able to implement a Web service using a back-end scripting language (Python Flask) and templating language (Jinja).
  14. Design and implement a relational database design with primary/foreign key constraints (SQLite).
  15. Interact with a database using Structured Query Language (SQL) to add, modify, and remove data records.

Problem sets are generally graded on three axes. They, along with some pertinent questions to consider, appear below:

  • Correctness: To what extent is your code consistent with the problem specifications and free of bugs? This is tested by comparing your code against a series of test cases.
  • Style: To what extent is your code readable (i.e., commented and intended with variables aptly named? Can I easily tell (via braces/indentation) the boundaries of each block of code?Is it clear what this variable does, based on its name? If I didn’t understand C code, would I be able to tell what is happening based on comments alone? Are there so few or so many comments that I am distracted by their presence or absence?
  • Design: To what extent is your code written well (i.e., clearly, efficiently, elegantly, and/or logically)? Can Repeated code be eliminated? Is there a “shorter” way to accomplish the same thing? Should some code have been broken out into its own function?

Specialization Tracks

After the first 15 weeks of the course, students are able to choose from one of three specialization tracks to learn more about coding. These tracks are based upon Harvard's own specialization courses. This material is presented as an additional educational experience for students, but has no direct impact on grading as there are no direct assignments for this portion of the course. Students are strongly encouraged to base their final projects on the material learned from these tracks.

The three tracks available to choose from are below. These tracks mirror the introduction to full courses which may be completed from HarvardX.

  • Web Programming with Python and JavaScript

    This track continues where the core CS50x Miami lectures leave off, diving more deeply into the design and implementation of web apps with Python, JavaScript, and SQL using frameworks like Flask, Django, and Bootstrap. Topics include more database design, scalability, security, and user experience.

  • Mobile App Development with React Native

    This track transitions from web development to mobile app development with React Native, a popular open-source framework maintained by Facebook that enables cross-platform native apps using JavaScript without Java or Swift. The track introduces students to modern JavaScript (including ES6 and ES7) as well as to JSX, a JavaScript extension.

  • Introduction to Game Development

    This track focuses on the development of 2D and 3D interactive games. Students explore the design of such childhood games as Super Mario Bros., Legend of Zelda, and Portal in a quest to understand how video games themselves are implemented. Via lectures and hands-on projects, the course explores principles of 2D and 3D graphics, animation, sound, and collision detection using frameworks like Unity and LÖVE 2D, as well as languages like Lua and C#.

Details about how to sign-up for the specific tracks will be given later on in the course.

Final Project

The climax of this course is its final project. The final project is your opportunity to take your newfound savvy with programming out for a spin and develop your very own piece of software. So long as your project draws upon this courses lessons, the nature of your project is entirely up to you, albeit subject to the staff's approval. All that we ask is that you build something of interest to you, that you solve an actual problem, that you impact campus, or that you change the world. Strive to create something that outlives this course.

That being said, there are some provisos. You may implement your project in any language(s) as long as the staff approves. You are welcome to utilize any infrastructure, provided the staff ultimately has access to any hardware and software that your project requires. The Idea Center has some hardware that may be used in some cases.

Inasmuch as software development is rarely a one-person effort, you are allowed an opportunity to collaborate with one or two classmates for this final project. Needless to say, it is expected that every student in any such group contribute equally to the design and implementation of that group’s project. Moreover, it is expected that the scope of a two- or three-person group’s project be, respectively, twice or thrice that of a typical one-person project. A one-person project, mind you, should entail more time and effort than is required by each of the course’s problem sets. Although no more than three students may design and implement a given project, you are welcome to solicit advice from others, so long as you respect the course’s policy on academic honesty.

The final project consists of 3 parts:

  • A project proposal. Students must present their projects in a closed-room presentation to the teaching staff. This presentation should be no more than 5 minutes in length and should describe what the individual or team wishes to accomplish in the final project. All projects must be pitched in this manner and approved before the project submission will be accepted for grading.
  • A project blog page. Each individual student (regardless if they are on a team) must create a blogging Website. This site must not utilize an existing content management system. A user authentication system should be implemented which allows a registered user to create, edit, or delete blog posts.
  • An individual or collaborative final project as described above. This project may utilize any content you learned from the core CS50x Miami material or the extended tracks near the end of the course.

Each component of the project will be graded and the respected weighted totals will be applied to the student's final grade.

Academic Honesty

For each graded assignment (quiz, p-set, project), you will be expected to provide attribution to any resources that you accessed which was not specifically given in the course lectures or sections. While building upon the publicly-available work of others is acceptable and encouraged, in no circumstances is it acceptable to do so and present it as your own. This includes attempting to pass off another student’s code as if it is your own. While it is encouraged that you share ideas with your classmates, you should never share code. If you are concerned that what you may be submitting violates standard academic dishonesty principles, ask a teaching staff member prior to submitting the assignment.

All assignments are subject to automated plagiarism detection upon submission. Significant evidence of plagiarism may result in receiving a score of 0 for the offending assignment and/or expulsion from the course for all parties involved.

You may opt to incriminate yourself in plagiarism and forgo the harshest penalties if contact heads@cs50xmiami.com with 48 hours of submitting the assignment and admit to violating this policy.
